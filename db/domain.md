## 데이모아 도메인

### 1. Auth

사용자가 시스템에 안전하게 로그인하고 인증 상태를 유지하는 책임을 가진 도메인  
로그인 / 로그아웃 / 토큰 / 세션

### 2. User

서비스를 사용하는 개인을 나타내는 기본 주체 도메인  
사용자 정보 (이름, 이메일 등)

### 3. Task

사용자가 수행해야 할 일정(할 일)을 정의하고 상태를 관리하는 도메인  
일정생성 / 일정수정 / 일정완료 / 일정삭제  

### 4. Category
일정을 의미 단위로 분류하기 위한 도메인
시스템 기본 카테고리 / 개인 카테고리 / 그룹 카테고리

### 5. Group

여러 사용자를 하나의 단위로 묶어 일정과 책임을 공유하기 위한 공간 도메인  
같이 쓰는 컨텍스트 제공

### 6. Membership

사용자와 그룹 사이의 소속 관계와 권한을 관리하는 도메인  
누가 어떤 그룹에 속해 있는지 / 누가 무엇을 할 수 있는지

### 7. GroupTask (또는 Task 확장)

그룹 안에서 특정 사용자에게 할 일을 할당하고 책임 관계를 명확히 하는 도메인
누가 누구에게 시켰는지  / 누가 수행해야 하는지

### 8. Notification

사용자에게 일정 변화나 요청을 적절한 시점에 알려주는 도메인
마감 알림 / 새로운 일정 등록 알림 / 할당 알림
